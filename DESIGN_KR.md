# Coralithm — 디자인 문서

## 컨셉

소셜 미디어 시청 기록을 살아있는 산호초로 시각화합니다.
사용자가 YouTube 데이터(Google Takeout)를 업로드하면 시청 패턴을 분석하여 각 주제 카테고리별로 고유한 산호를 생성합니다.
각 산호의 형태, 크기, 색상, 발광 강도, 움직임은 실제 데이터에 의해 결정됩니다 — 같은 산호초는 존재하지 않습니다.

---

## 현재 구현

### 데이터 입력 (두 가지 모드)

**모드 A — Google Takeout 업로드 (실제 데이터)**
- "📂 YouTube 데이터 업로드" 버튼 클릭 → Google Takeout에서 내보낸 `watch-history.json` 선택
- 프론트엔드 전용 파싱 (서버 없음, 데이터 외부 전송 없음)
- 추출 항목: 동영상 제목, 채널명, 타임스탬프
- 분류: 14개 주제 카테고리에 대한 키워드 매칭 (한/영/중 다국어 키워드)
- 카테고리별 산출: 시청 횟수, 채널 다양성, 최근 활성도, 트렌드(관심 증가/감소)
- 사용자 데이터에서 발견된 카테고리별로 고유한 시각 속성을 가진 산호 생성

**모드 B — 수동 선택 (데모/대체)**
- "+ 산호 추가" 버튼 클릭 → 14개 카테고리 태그 모달 → 3개 선택 → 시뮬레이션 데이터로 산호 생성
- 선택 순서가 가중치 결정: 1번째 = 50%, 2번째 = 30%, 3번째 = 20%

**튜토리얼**
- 우하단 "?" 버튼 → Google Takeout에서 YouTube 데이터를 내보내는 5단계 가이드
- takeout.google.com 직접 링크 포함
- 개인정보 안내: 데이터는 브라우저에서만 처리됨

### 데이터 → 시각 매핑 (7개 차원)

각 복제 산호는 데이터에 따라 시각적으로 고유합니다. 모든 매핑은 표준 Three.js API 사용, 커스텀 셰이더 없음.

| 데이터 차원 | 시각 파라미터 | 지각적 의미 |
|---|---|---|
| 카테고리 | 색조 (색상) | 다른 관심사 = 다른 색상 |
| 시청 비중 (가중치) | Y축 스케일 (높이) | 많이 봤을수록 = 높은 산호 |
| 채널 다양성 | XZ 스케일 (두께) | 다양한 관심사 = 두꺼운 산호 |
| 최근 활성도 | 자체발광 강도 | 최근 시청 = 밝게 빛남; 오래 안 봄 = 어둡게 |
| 시청 빈도 트렌드 | 회전 속도 | 관심 증가 = 빠른 회전 |
| 활성 수준 | 호흡 주파수 | 활발 = 빠른 맥동 |
| 가중치 + 노이즈 시드 | 심플렉스 노이즈 정점 변형 | 각 산호의 실루엣이 미묘하게 다름 |

**정점 변형**은 복제 시 심플렉스 노이즈로 한 번 적용됩니다. 진폭은 시청량에 비례 — 시청이 많은 카테고리일수록 더 왜곡되고 유기적인 형태가 됩니다.

### 시각 분위기

- 순수 검정 배경 (#000000)
- UnrealBloomPass 블룸 후처리
- 지수 안개 (밀도 0.04)
- CSS 비네트 오버레이
- 3000개 환경 파티클 흐름 (가산 블렌딩 표류)
- 800개 마린 스노우 파티클 (하강 + 수평 흔들림)
- 3개 빛줄기 (흔들림 + 명멸)

### 인터랙션

- **오빗 컨트롤**: 드래그로 회전, 스크롤로 줌
- **산호 클릭 → 카메라 비행 포커스**: 부드러운 이징, 상세 패널 표시 (카테고리명, 비중 %, 시청 수, 채널 다양성, 활성도)
- **빈 곳 클릭 → 전체 뷰 복귀**
- **다른 산호 페이드**: 포커스되지 않은 산호는 70%로 축소, 불투명도 12%로 감소
- **산호 삭제**: 상세 패널의 "산호 삭제" 버튼 → 산호가 0으로 축소되어 사라지고 연결선 자동 재구성
- **CSS2D 플로팅 라벨**: 각 산호 위에 카테고리명, 거리에 따라 페이드
- **성장 애니메이션**: 생성 시 0에서 탄성 있게 성장 (easeOutBack)
- **자동 배치**: 나선형 알고리즘 + 충돌 회피, 새 산호가 겹치지 않음

### 연결선

- 2차 베지어 곡선으로 각 산호와 가장 가까운 2개 이웃 연결
- 가산 블렌딩, 호흡 불투명도 애니메이션
- 산호 추가/삭제 시 동적 재구성
- 모든 산호 성장 완료 후 나타남

### 기술 스택

- Vite (바닐라 JS, 프레임워크 없음)
- Three.js r184 (WebGLRenderer, EffectComposer, UnrealBloomPass, OutputPass, CSS2DRenderer)
- THREE.Timer
- simplex-noise (정점 변형)
- 단일 파일: `src/main.js`

### 모델

- `coral.glb` (Tripo AI 생성, 17 MB, glTF 2.0, 비압축)
- 메쉬 1개, PBR 머티리얼 1개 (베이스컬러 + 러프니스/메탈릭 + 노말)
- 산호마다 독립 복제, 독립 머티리얼 속성 및 정점 변형 적용

### 배포

- Vercel 정적 사이트: **sn-rouge.vercel.app**
- `git push` 시 자동 재배포
- 서버 불필요; 모든 처리는 브라우저에서 완료

## 예정 기능 (구현 가능)

### 데이터 파이프라인 강화 (3계층 분류)

현재: 키워드 매칭만 (2계층). 정확도 향상:

**1계층 — YouTube Data API v3 + 룩업 테이블 (약 80% 커버, AI 비용 제로)**
- 백엔드에서 Takeout URL의 동영상 ID로 YouTube API 호출
- 각 동영상에 `categoryId` 존재 (Gaming=20, Music=10, Education=27…)
- 정적 룩업 테이블로 categoryId → Coralithm 주제 매핑
- OAuth / API 키 관리를 위한 경량 백엔드 필요

**2계층 — 키워드 매칭 (현재 구현, 약 10-15% 커버)**
- 구현 완료, 14개 카테고리 × 6-10개 키워드 (한/영/중)

**3계층 — LLM 스마트 분석 (나머지 약 5-10% + 감정 추론)**
- 모호한 제목 → Claude/GPT에 배치 전송하여 의미 분류
- 감정 추론: 카테고리별 콘텐츠 소비 성향 분석
- 배치 호출: 동영상별이 아닌 사용자별 1회

### 비주얼 업그레이드

- **다종 산호 모델**: 카테고리별 다른 기하학적 형태 — 모델링 소프트웨어 또는 AI 3D 도구로 추가 제작 필요
- **성장 변형 애니메이션**: Shape Keys / Morph Targets로 가지 확장 — 3D 소프트웨어에서 제작 필요
- **군락별 파티클 헤일로**: 각 산호 주위 독립적인 컬러 파티클 구름
- **선택적 블룸**: 자체발광 요소만 글로우, 비발광 표면은 선명 유지

### 인터랙션 업그레이드

- **호버 하이라이트**: 마우스 오버 시 산호 밝아짐 + 라벨 확대
- **하위 카테고리 전개**: 산호 클릭 후 하위 주제 표시 (예: Lifestyle → 요리 / 패션 / 운동)
- **스크린샷 내보내기**: 산호초를 PNG로 저장
