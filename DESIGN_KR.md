# Coralithm — 디자인 문서

## 컨셉

YouTube 시청 기록을 산호초로 시각화합니다.
사용자가 Google Takeout 데이터를 업로드하면 시청 기록을 주제별로 분류하고 카테고리마다 산호를 하나씩 생성합니다. 산호의 크기, 색상, 발광, 움직임은 해당 카테고리를 얼마나, 얼마나 최근에, 얼마나 다양하게 봤는지를 반영합니다.

---

## 현재 구현

### 데이터 입력

**Google Takeout 업로드**
- Google Takeout에서 내보낸 `watch-history.json` 파일 선택
- 브라우저에서만 처리, 서버 전송 없음
- 동영상 제목, 채널명, 타임스탬프 추출
- 14개 주제 카테고리에 키워드 매칭으로 분류 (한/영)
- 카테고리별: 시청 수, 채널 다양성, 최근 활성도, 트렌드 산출

**수동 선택 (데모용)**
- 모달에서 카테고리 3개 선택, 시뮬레이션 데이터로 산호 생성
- 선택 순서로 가중치 결정: 1번째 50%, 2번째 30%, 3번째 20%

**튜토리얼**
- 우하단 "?" 버튼으로 Google Takeout 내보내기 가이드 열기, takeout.google.com 직접 링크 포함

### 데이터 → 시각 매핑

각 산호는 같은 베이스 모델의 복제이지만 7개 데이터 차원이 외형을 다르게 만듭니다:

| 데이터 | 매핑 대상 | 이유 |
|---|---|---|
| 카테고리 | 색상 | 한눈에 구분 |
| 시청 수 | 높이 (Y 스케일) | 많이 봤으면 높게 |
| 채널 다양성 | 두께 (XZ 스케일) | 폭넓게 봤으면 두껍게 |
| 최근 활성도 | 자체발광 밝기 | 최근 시청 = 밝게, 오래됨 = 어둡게 |
| 빈도 트렌드 | 회전 속도 | 관심 증가 = 빠르게 회전 |
| 활성 수준 | 호흡 주기 | 활발 = 빠른 맥동 |
| 가중치 × 노이즈 | 정점 변형 (simplex noise) | 복제 느낌 제거 |

정점 변형은 산호 생성 시 한 번 적용됩니다. 시청량이 많을수록 변형이 커집니다.

### 분위기

- 검정 배경, 블룸, 지수 안개, 비네트
- 3000개 표류 파티클, 800개 마린 스노우, 3개 빛줄기

### 인터랙션

- 드래그/스크롤로 회전, 줌
- 산호 클릭: 카메라 비행 포커스, 상세 패널 표시, 나머지 산호 페이드아웃
- 빈 곳 클릭: 카메라 복귀, 전체 복원
- 상세 패널에 삭제 버튼, 삭제 시 연결선 자동 재구성
- 라벨은 산호 위에 떠 있고 거리에 따라 페이드
- 새 산호는 0에서 탄성 성장, 나선형 배치로 겹침 방지

### 연결선

- 베지어 곡선으로 가장 가까운 2개 이웃과 연결
- 성장 후 나타남, 추가/삭제 시 재구성

### 기술

- Vite, 바닐라 JS, 단일 파일 `src/main.js`
- Three.js r184, CSS2DRenderer, UnrealBloomPass, simplex-noise
- 모델: `coral.glb` (Tripo, 17 MB, PBR)
- 배포: Vercel **sn-rouge.vercel.app**, push 시 자동 배포

## 향후 방향

### 분류 정확도 개선

현재 키워드 매칭만 사용 중. 단계적으로 보강 가능:

1. **YouTube API**: Takeout URL에서 추출한 동영상 ID로 `categoryId` 조회. 대부분의 동영상 커버. API 키 관리용 소규모 백엔드 필요.
2. **키워드 폴백**: 이미 구현됨, API가 못 잡는 부분 보완.
3. **LLM 배치**: 모호한 제목 의미 분류 + 카테고리별 감정 태깅. 사용자당 1회 호출.

### 시각적 다양화

- 카테고리별 다른 산호 모델 (추가 glb 파일 필요)
- Shape Keys로 가지 성장 애니메이션 (모델링 작업 필요)
- 산호별 개별 컬러 파티클
- 레이어 기반 선택적 블룸

### 인터랙션 확장

- 호버 프리뷰
- 하위 카테고리 드릴다운
- 스크린샷 내보내기
