# Coralithm — 디자인 문서

## 컨셉

소셜 미디어 데이터를 살아있는 산호초로 시각화합니다.
각 산호 군락은 하나의 주제 카테고리를 나타내며, 크기·색상·감정은 사용자 데이터에 의해 결정됩니다.
산호초는 실시간으로 성장하여 사용자마다 고유한 "디지털 정체성 초상화"를 형성합니다.

---

## 현재 구현 (프로토타입)

### 핵심

- **7개의 산호 군락**이 밀집된 산호초로 배치되며, 각각 주제 카테고리별 색상 적용
- **데이터 기반 크기 조절**: 각 군락의 스케일은 `weight` 값에 비례
- **목업 데이터**: 주제 / 감정 / 가중치가 `main.js` 상단에 JSON 배열로 정의되어 있으며, API 응답으로 쉽게 교체 가능
- **성장 애니메이션**: 페이지 로드 시 각 군락이 순서대로 0에서부터 성장 (easeOutBack 이징), 연결선은 모든 군락 성장 완료 후 페이드인

### 비주얼

- **생물발광 분위기**: 순수 검정 배경, Bloom 후처리 효과, 지수 안개, 비네트 오버레이
- **해양설(마린 스노우)**: 800개의 미립자가 천천히 하강하며 약간의 수평 흔들림
- **틴들 광선**: 3개의 반투명 평면이 위에서 내리쬐는 빛줄기를 시뮬레이션하며, 천천히 흔들리고 명멸
- **자체발광 호흡**: 각 군락의 발광 강도가 시간에 따라 부드럽게 맥동
- **환경 파티클 흐름**: 3000개의 주변 파티클이 산호초 주위를 떠다님 (가산 블렌딩)

### 인터랙션

- **오빗 컨트롤**: 드래그로 회전, 스크롤로 줌
- **산호 클릭 → 카메라 비행 포커스**: 부드러운 easeInOut 트윈으로 클릭한 군락에 포커스; 좌하단에 상세 패널 표시 (주제, 감정, 비율 %)
- **빈 곳 클릭 → 전체 뷰 복귀**: 카메라가 초기 위치로 비행
- **CSS2D 플로팅 라벨**: 각 군락 위에 주제명이 떠 있으며, 거리에 따라 자동 페이드, 다른 군락 포커스 시 어두워짐

### 연결선

- 곡선 (2차 베지어)으로 각 군락과 가장 가까운 2개 이웃을 연결
- 가산 블렌딩, 낮은 불투명도, 명멸 애니메이션
- 성장 애니메이션 완료 후에만 나타남

### 기술 스택

- Vite (바닐라 JS, 프레임워크 없음)
- Three.js r184 (WebGLRenderer, EffectComposer, UnrealBloomPass, OutputPass, CSS2DRenderer)
- THREE.Timer (더 이상 사용되지 않는 Clock 대체)
- 단일 파일: `src/main.js` (약 420줄)

### 모델

- `coral.glb` (Tripo AI 생성, 17 MB, glTF 2.0, 비압축)
- 메쉬 1개, PBR 머티리얼 1개 (베이스컬러 + 러프니스/메탈릭 + 노말 텍스처), 애니메이션 1개 (미사용)
- 각 군락마다 클론 후 고유한 자체발광 색상 오버레이

## 예정 기능 (구현 가능)

### 데이터 파이프라인

- **입력 페이지**: 사용자가 YouTube 채널 URL을 붙여넣기 (또는 연결된 플랫폼 선택)
- **YouTube Data API v3** (무료 할당량): 구독 채널 가져오기 → 채널 카테고리를 주제 군락에 매핑 → 구독 수 / 시청 시간으로 가중치 계산
- 출력: `[{topic, emotion, weight}, ...]` → 현재의 `CLUSTERS` 배열에 직접 대입
- 감정 매핑: 콘텐츠 카테고리 기반 휴리스틱 추론 (예: Gaming → 스릴, Music → 흥분) — ML이 아닌 룩업 테이블; 데모에 충분

### 비주얼 업그레이드

- **다종 산호 모델**: 주제별로 다른 기하학적 형태 (가지형 / 뇌형 / 판형 / 녹각형), 동일 glb를 7번 클론하는 대신 — 추가 모델 조달 또는 생성 필요
- **군락별 파티클 헤일로**: 각 군락 색상에 맞춘 작은 파티클 구름이 개별 군락 주위를 공전
- **선택적 블룸**: 자체발광 요소만 글로우 적용 (레이어 + 별도 블룸 패스 사용), 비발광 표면은 선명하게 유지

### 인터랙션 업그레이드

- **호버 하이라이트**: 마우스 오버 시 군락 밝아짐 + 라벨 확대 (클릭 전 피드백)
- **상세 패널 확장**: 각 주제 하위에 구독 상위 3개 채널 / 가장 많이 본 콘텐츠 표시
- **공유 / 내보내기**: 스크린샷 버튼 (renderer.domElement.toDataURL)으로 산호초를 PNG로 저장

### 배포

- Vercel 정적 사이트 (서버 불필요)
- `npm run build` → `dist/` 그대로 서빙
- glb는 `public/models/`에서 제공 (빌드 산출물에 포함)
