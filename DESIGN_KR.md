# Coralithm — 디자인 문서

## 컨셉

Coralithm은 한 사람의 관심사를 계속 자라나는 3D 산호초로 바꾸는 프로젝트입니다.

사용자는 YouTube 시청 기록을 업로드하거나 짧은 질문에 답할 수 있습니다. 두 방식 모두 관심 카테고리와 비율을 만들고, 그 결과가 산호의 색, 형태, 밝기, 움직임에 반영됩니다. 질문을 다시 풀면 기존 산호초를 지우지 않고 새 산호를 더할 수 있습니다.

---

## 현재 구현

### 두 가지 입력 방식

**YouTube 데이터 업로드**

- Google Takeout에서 내보낸 `watch-history.json`을 읽습니다.
- 파일은 브라우저 안에서만 처리하며 시청 기록을 서버로 보내지 않습니다.
- 영상 제목, 채널명, 시청 시간을 추출합니다.
- 한글과 영문 키워드로 기록을 14개 주제에 분류합니다.
- 카테고리별 시청 수, 채널 수, 최근 시청 시점, 트렌드를 계산합니다.
- 시청 수가 많은 상위 10개 카테고리로 산호를 만듭니다.
- 새 파일을 업로드하면 기존 산호초를 지우고 다시 구성합니다.

**랜덤 질문 모드**

- 총 14문제 중 매번 겹치지 않게 3문제를 뽑습니다.
- 각 문제에는 4개의 답변이 있고, 답변마다 하나의 관심 카테고리가 연결되어 있습니다.
- 구체적인 행동이나 실제 선택을 묻는 질문은 가중치가 높고, 감정이나 추상적 취향을 묻는 질문은 낮습니다.
- 같은 카테고리가 여러 번 나오면 점수가 누적됩니다.
- 최종 점수 비율로 하나의 복합 관심 산호를 만듭니다.
- 질문을 반복해서 풀면 기존 산호초에 새 산호가 추가됩니다.
- 이름은 적중한 카테고리를 조합해 자동으로 붙입니다. 예: `게임·음악 산호`

데모용 수동 선택 기능도 남아 있습니다. 카테고리 3개를 고르면 선택 순서에 따라 50%, 30%, 20%의 고정 가중치를 사용합니다.

### 카테고리

현재 카테고리는 14개입니다.

뉴스·시사, 경제·투자, 요리, 게임, 스포츠, 소프트웨어·AI, 환경·기후, 광고·마케팅, 음악, 디자인·예술, 여행, 영감·인사이트, 학습, 스타일.

YouTube 기록에서 어느 카테고리에도 맞지 않는 항목은 `기타`로 분류합니다.

### 데이터와 시각 표현

현재 7개의 산호 모델을 불러와 주 카테고리에 따라 순환 배정합니다. 카테고리가 모델보다 많기 때문에 일부 카테고리는 같은 기본 형태를 공유합니다.

| 데이터 | 시각 표현 |
|---|---|
| 주 카테고리 | 산호 색과 기본 모델 |
| 카테고리 비율 | 높이, 투명도, 정점 변형 강도 |
| 채널 다양성 | 가로 두께와 재질 거칠기 |
| 최근 활성도 | 자체발광과 호흡 속도 |
| 트렌드 | 회전 속도 |
| 여러 적중 카테고리 | 상세 패널의 비율 목록과 조합 이름 |

산호를 만들 때 simplex noise로 정점을 한 번 변형해 실루엣에 차이를 줍니다. 새 산호는 거의 0인 크기에서 탄성 있게 자라며, 기존 산호와 겹치지 않는 위치를 자동으로 찾습니다.

### 질문 모드의 시뮬레이션 수치

질문 답변으로 알 수 있는 것은 관심 카테고리와 상대적인 비율입니다. 실제 시청 수, 채널 다양성, 최근 시청 활동까지 알 수는 없습니다.

현재 구현은 기존 시각 시스템을 작동시키기 위해 다음 수치를 임시로 만듭니다.

- 시청 수: 가장 높은 카테고리 비율로 환산
- 채널 다양성: 적중한 카테고리 수로 추정
- 최근 활성도: 가장 높은 카테고리 비율로 환산
- 트렌드: 고정값 사용

상세 패널에서는 이 값과 YouTube 데이터에서 계산한 실제 통계를 아직 구분하지 않습니다. 두 모드가 같은 근거를 가진 것처럼 보일 수 있으므로, 정식 버전에서는 질문 모드의 해당 항목을 숨기거나 `추정`으로 표시하거나 질문에 맞는 다른 지표로 바꿔야 합니다.

### 장면과 인터랙션

- 검은 수중 배경, 블룸, 지수 안개, 비네트
- 표류 파티클 3,000개, 마린 스노우 800개, 빛줄기 3개
- 드래그로 회전하고 스크롤로 확대·축소
- 산호에 마우스를 올리면 라벨 강조
- 산호를 클릭하면 카메라가 가까이 이동하고 다른 산호는 흐려지며 상세 패널이 열림
- 빈 곳을 클릭하면 전체 화면으로 복귀
- 개별 산호 삭제 가능
- 각 산호를 가장 가까운 2개의 이웃과 연결하고, 추가·삭제 시 연결선 재구성
- 현재 화면을 PNG로 저장
- `?` 버튼에서 Google Takeout 내보내기 방법 안내

## 현재 한계

### YouTube 분류가 아직 단순함

현재는 제목의 키워드만 봅니다. 제목이 모호하거나 키워드가 없거나 여러 주제에 걸치면 잘못 분류될 수 있고, 인식하지 못한 기록이 `기타`에 많이 모일 수 있습니다.

### 트렌드 지표를 다시 정의해야 함

트렌드 필드는 이미 있고 회전 속도에 연결되어 있지만, 현재 계산 방식만으로는 관심사가 시간에 따라 어떻게 변했는지 제대로 보여주기 어렵습니다. 비교할 기간과 계산 방식을 새로 정해야 합니다.

### 두 모드는 서로 다른 데이터를 다룸

YouTube 모드는 실제 시청 행동을 보여주고, 질문 모드는 사용자가 직접 고른 현재의 취향을 보여줍니다. 같은 산호 표현을 사용할 수는 있지만 상세 패널에서 둘을 같은 종류의 측정값처럼 다루면 안 됩니다.

### 모델과 카테고리가 일대일로 대응하지 않음

14개 카테고리가 7개 모델을 공유합니다. 지금은 색상이 카테고리 구분의 큰 부분을 맡고 있으며, 주제마다 고유한 형태를 갖는 단계는 아닙니다.

## 다음 방향

### 데이터의 의미부터 정리

- 실제 통계와 질문 기반 추정을 명확히 표시
- 질문 모드에는 `관심 강도`, `취향 집중도`처럼 답변으로 설명 가능한 지표 사용
- 단순 카테고리 조합을 넘어서는 산호 이름 방식 마련
- 트렌드 계산 방식 재설계

### 분류 정확도 개선

1. Takeout URL에서 영상 ID를 추출해 YouTube API의 `categoryId` 조회
2. 현재 키워드 분류는 폴백으로 유지
3. 모호한 제목은 묶어서 의미 기반으로 분류

### 시각적 차이 확장

- 주요 카테고리가 더 뚜렷한 실루엣을 갖도록 모델 추가
- Shape Keys나 리깅을 이용한 가지 성장
- 산호마다 다른 파티클과 국소 발광 적용
- 카테고리 구성과 데이터 출처를 더 분명하게 보여주는 화면 추가
