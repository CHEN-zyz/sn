# Coralithm — 设计文档

## 概念

将社交媒体数据可视化为一片活的珊瑚礁。
每簇珊瑚代表一个主题类目；大小、颜色和情绪由用户数据驱动。
珊瑚礁实时生长，为每个用户生成独一无二的"数字身份画像"。

---

## 当前实现（原型）

### 核心

- **7 簇珊瑚** 紧密排列成一片礁体，每簇按主题类目着色
- **数据驱动大小**：每簇的缩放比例与其 `weight` 权重值成正比
- **模拟数据**：主题 / 情绪 / 权重以 JSON 数组定义在 `main.js` 顶部，可直接替换为 API 返回值
- **生长动画**：页面加载后，各簇按顺序依次从无到有生长（easeOutBack 缓动），连接线在全部生长完成后淡入

### 视觉

- **生物发光氛围**：纯黑背景、Bloom 后处理辉光、指数雾、暗角遮罩
- **海雪**：800 颗微粒缓缓下沉，带轻微横向摆动
- **丁达尔光束**：3 片半透明平面模拟从上方投下的光柱，缓慢摇曳和呼吸
- **自发光呼吸**：每簇的发光强度随时间柔和脉动
- **环绕粒子流**：3000 颗环境粒子在礁体周围漂流（加法混合）

### 交互

- **轨道控制**：拖拽旋转、滚轮缩放
- **点击珊瑚 → 相机飞入聚焦**：平滑 easeInOut 缓动飞向被点击的簇；左下角弹出详情面板（主题、情绪、占比 %）
- **点击空白 → 返回总览**：相机飞回初始位置
- **CSS2D 浮动标签**：主题名悬浮于每簇上方，按距离自动淡出，聚焦其他簇时压暗

### 连接线

- 曲线（二次贝塞尔）连接每簇与其最近的 2 个邻居
- 加法混合、低透明度、呼吸动画
- 在生长动画完成后才出现

### 技术栈

- Vite（原生 JS，无框架）
- Three.js r184（WebGLRenderer、EffectComposer、UnrealBloomPass、OutputPass、CSS2DRenderer）
- THREE.Timer（替代已弃用的 Clock）
- 单文件：`src/main.js`（约 420 行）

### 模型

- `coral.glb`（Tripo AI 生成，17 MB，glTF 2.0，未压缩）
- 1 个网格、1 个 PBR 材质（底色贴图 + 粗糙度/金属度 + 法线贴图）、1 个动画（未使用）
- 每簇克隆后叠加独立的自发光颜色

## 预期功能（可实现）

### 数据接入（三层分类管线）

**第一层 — YouTube API + 查表映射（覆盖约 80%，零 AI 成本）**
- 用户粘贴 YouTube 频道 URL → 后端调用 YouTube Data API v3（免费额度）
- 拉取订阅/观看记录；每个视频/频道自带 YouTube `categoryId`（Gaming=20、Music=10、Education=27…）
- 静态查表将 YouTube categoryId → Coralithm 主题簇
- 按每个簇的订阅数/观看时长计算权重

**第二层 — 关键词/规则兜底（覆盖约 10%）**
- 被归为笼统类目（如"Entertainment"、"People & Blogs"）的视频，对标题+描述做关键词匹配
- 规则示例：标题含"食谱"/"烹饪" → Lifestyle；含"教程"/"编程" → Tech
- 确定性、快速、无 API 成本

**第三层 — LLM 智能分析（覆盖剩余约 10% + 情绪推断）**
- 经过第一、二层仍未明确分类的内容 → 将标题+描述批量发送给 LLM（Claude / GPT）做主题分类
- **情绪推断**（LLM 的核心价值）：分析用户的内容消费模式，判断每个主题下的情绪倾向（如该用户的 Gaming 内容偏"刺激"还是"治愈"）——关键词无法做到
- 批量调用以降低 API 成本（每用户一次调用，而非每视频一次）

**输出**：`[{topic, emotion, weight}, ...]` → 直接替换前端 `CLUSTERS` 数组

### 视觉升级

- **多种珊瑚模型**：不同主题对应不同几何形态（枝状 / 脑状 / 板状 / 鹿角状），而非克隆同一个 glb——需要额外获取或生成模型
- **簇级粒子光晕**：每簇独立的小粒子云，着色与簇颜色一致，围绕各自簇体运动
- **选择性辉光**：仅自发光元素产生辉光（通过图层 + 独立 Bloom Pass），保持非发光表面清晰

### 交互升级

- **悬停高亮**：鼠标悬停时簇变亮、标签放大（点击前的预反馈）
- **详情面板扩展**：每个主题下显示前 3 个订阅频道 / 最常观看内容
- **分享/导出**：截图按钮（renderer.domElement.toDataURL），将珊瑚礁保存为 PNG

### 部署

- Vercel 静态站（无需服务器）
- `npm run build` → `dist/` 直接服务
- glb 从 `public/models/` 提供（包含在构建产物中）
