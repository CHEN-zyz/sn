# Coralithm — 设计文档

## 概念

将社交媒体观看历史可视化为一片活的珊瑚礁。
用户上传 YouTube 数据（Google Takeout），系统分析观看模式，为每个主题类目生成一株独特的珊瑚。
每株珊瑚的形态、大小、颜色、发光强度和运动方式都由真实数据驱动——没有两片珊瑚礁长得一样。

---

## 当前实现

### 数据输入（两种模式）

**模式 A — Google Takeout 上传（真实数据）**
- 用户点击"📂 YouTube 데이터 업로드"→ 选择 Google Takeout 导出的 `watch-history.json`
- 纯前端解析（无服务器，数据不上传至外部）
- 提取：视频标题、频道名、时间戳
- 分类：基于 14 个主题类目的关键词匹配（中/英/韩三语关键词）
- 计算每个类目的：观看次数、频道多样性、最近活跃度、趋势（兴趣上升/下降）
- 为用户数据中出现的每个类目生成一株珊瑚，每株具有独特的视觉属性

**模式 B — 手动选择（演示/备用）**
- 用户点击"+ 산호 추가"→ 弹窗显示 14 个类目标签 → 选择 3 个 → 生成带模拟数据的珊瑚
- 选择顺序决定权重：第 1 个 = 50%，第 2 个 = 30%，第 3 个 = 20%

**教程**
- 右下角"?"按钮 → 弹出 5 步指引，教用户如何从 Google Takeout 导出 YouTube 数据
- 包含 takeout.google.com 直达链接
- 隐私声明：数据仅在浏览器端处理

### 数据 → 视觉映射（7 个维度）

每株克隆珊瑚的视觉根据其数据独一无二。所有映射使用标准 Three.js API，无自定义着色器。

| 数据维度 | 视觉参数 | 感知含义 |
|---|---|---|
| 类目 | 色相（颜色） | 不同兴趣 = 不同颜色 |
| 观看占比（权重） | Y 轴拉伸（高度） | 看得多 = 长得高 |
| 频道多样性 | XZ 缩放（粗细） | 涉猎广 = 长得粗 |
| 最近活跃度 | 自发光强度 | 最近在看 = 亮；很久没看 = 暗 |
| 观看频率趋势 | 自转速度 | 兴趣上升 = 转得快 |
| 活跃度 | 呼吸频率 | 活跃 = 快速脉动 |
| 权重 + 噪声种子 | simplex noise 顶点变形 | 每株珊瑚轮廓微妙不同 |

**顶点变形**在克隆时一次性应用 simplex noise。振幅随观看量缩放——重度类目产生更扭曲、有机的形态。

### 视觉氛围

- 纯黑背景（#000000）
- UnrealBloomPass 辉光后处理
- 指数雾（密度 0.04）
- CSS 暗角遮罩
- 3000 环境粒子流（加法混合漂流）
- 800 海雪粒子（下坠 + 横向摆动）
- 3 条丁达尔光束（摇曳 + 明灭）

### 交互

- **轨道控制**：拖拽旋转、滚轮缩放
- **点击珊瑚 → 相机飞入聚焦**：平滑缓动，弹出详情面板（类目名、占比 %、观看次数、频道多样性、活跃度）
- **点击空白 → 返回总览**
- **其他珊瑚淡化**：未聚焦的珊瑚缩至 70% 并降至 12% 不透明度
- **删除珊瑚**：详情面板中"산호 삭제"按钮 → 珊瑚缩回为零消失，连接线自动重建
- **CSS2D 浮动标签**：类目名悬浮于珊瑚上方，按距离淡出
- **生长动画**：每株珊瑚生成时从零弹出（easeOutBack）
- **自动避让摆位**：螺旋算法 + 碰撞检测，新珊瑚不会重叠

### 连接线

- 二次贝塞尔曲线连接每株珊瑚与最近的 2 个邻居
- 加法混合，呼吸式透明度动画
- 添加或删除珊瑚时动态重建
- 所有珊瑚生长完成后才出现

### 技术栈

- Vite（原生 JS，无框架）
- Three.js r184（WebGLRenderer、EffectComposer、UnrealBloomPass、OutputPass、CSS2DRenderer）
- THREE.Timer
- simplex-noise（顶点变形）
- 单文件：`src/main.js`

### 模型

- `coral.glb`（Tripo AI 生成，17 MB，glTF 2.0，未压缩）
- 1 个网格、1 个 PBR 材质（底色 + 粗糙度/金属度 + 法线贴图）
- 每株珊瑚独立克隆，具有独立材质属性和顶点变形

### 部署

- Vercel 静态站：**sn-rouge.vercel.app**
- 每次 `git push` 自动重新部署
- 无需服务器；所有处理在浏览器端完成

## 预期功能（可实现）

### 数据管线增强（三层分类）

当前：仅关键词匹配（第二层）。提升准确率：

**第一层 — YouTube Data API v3 + 查表（覆盖约 80%，零 AI 成本）**
- 后端用 Takeout URL 中提取的视频 ID 调用 YouTube API
- 每个视频自带 `categoryId`（Gaming=20、Music=10、Education=27…）
- 静态查表映射 categoryId → Coralithm 主题
- 需要轻量后端做 OAuth / API key 管理

**第二层 — 关键词匹配（当前实现，覆盖约 10-15%）**
- 已实现，14 个类目 × 6-10 个关键词（中英韩三语）

**第三层 — LLM 智能分析（剩余约 5-10% + 情绪推断）**
- 模糊标题 → 批量发送给 Claude/GPT 做语义分类
- 情绪推断：分析用户每个类目下的内容消费倾向
- 批量调用：每用户一次，而非每视频一次

### 视觉升级

- **多种珊瑚模型**：不同类目对应不同几何形态——需要建模软件或 AI 3D 工具额外制作
- **生长变形动画**：Shape Keys / Morph Targets 实现枝干延伸——需要在 3D 软件中制作
- **簇级粒子光晕**：每株珊瑚周围独立的彩色粒子云
- **选择性辉光**：仅自发光元素产生辉光，非发光表面保持清晰

### 交互升级

- **悬停高亮**：鼠标悬停时珊瑚变亮、标签放大
- **子分类展开**：点入珊瑚后显示子主题（如 Lifestyle → 料理 / 时尚 / 运动）
- **截图导出**：保存珊瑚礁为 PNG
