# Coralithm — 设计文档

## 概念

将 YouTube 观看历史可视化为珊瑚礁。
用户上传 Google Takeout 数据，系统按主题类目分类观看记录，为每个类目生成一株珊瑚。珊瑚的大小、颜色、发光和动态反映用户在该类目下看了多少、看了多久、涉猎多广。

---

## 当前实现

### 数据输入

**Google Takeout 上传**
- 用户选择 Google Takeout 导出的 `watch-history.json`
- 浏览器端解析，不上传任何数据到服务器
- 提取视频标题、频道名、时间戳
- 按 14 个主题类目做关键词匹配分类（中/英/韩）
- 每个类目算出：观看次数、频道多样性、最近活跃度、趋势

**手动选择（演示用）**
- 从弹窗选 3 个类目，用模拟数据生成珊瑚
- 选择顺序决定权重：第 1 个 50%，第 2 个 30%，第 3 个 20%

**教程**
- 右下角"?"按钮打开导出指引，带 takeout.google.com 链接

### 数据 → 视觉映射

每株珊瑚是同一个基础模型的克隆，但 7 个数据维度让它们看起来不一样：

| 数据 | 映射到 | 为什么 |
|---|---|---|
| 类目 | 颜色 | 一眼区分不同兴趣 |
| 观看次数 | 高度（Y 轴） | 看得多 = 长得高 |
| 频道多样性 | 粗细（XZ 轴） | 涉猎广 = 长得粗 |
| 最近活跃度 | 自发光亮度 | 最近在看 = 亮，很久没看 = 暗 |
| 频率趋势 | 自转速度 | 兴趣上升 = 转得快 |
| 活跃度 | 呼吸频率 | 活跃 = 脉动快 |
| 权重 × 噪声 | 顶点变形（simplex noise） | 打破"复制粘贴"的轮廓 |

顶点变形在珊瑚创建时做一次。观看量越大，变形越明显。

### 氛围

- 纯黑背景、辉光后处理、指数雾、暗角
- 3000 漂流粒子、800 海雪、3 条光束

### 交互

- 拖拽旋转、滚轮缩放
- 点击珊瑚：相机飞入，弹出详情面板，其他珊瑚淡化
- 点击空白：相机飞回，全部恢复
- 详情面板里有删除按钮，删除后连接线自动重建
- 标签悬浮在珊瑚上方，距离远了自动淡出
- 新珊瑚从零弹出生长，自动摆位避免重叠

### 连接线

- 贝塞尔曲线连接每株珊瑚与最近的 2 个邻居
- 生长完成后淡入，增删珊瑚时重建

### 技术

- Vite，原生 JS，单文件 `src/main.js`
- Three.js r184、CSS2DRenderer、UnrealBloomPass、simplex-noise
- 模型：`coral.glb`（Tripo，17 MB，标准 PBR）
- 部署在 Vercel：**sn-rouge.vercel.app**，push 自动更新

## 后续方向

### 分类准确率提升

目前只有关键词匹配，可以分层加强：

1. **YouTube API**：用 Takeout URL 里的视频 ID 查 `categoryId`，覆盖大部分视频。需要小后端管理 API key。
2. **关键词兜底**：已做，补漏 API 没覆盖的。
3. **LLM 批量处理**：模糊标题的语义分类 + 每个类目的情绪标签。每用户一次调用。

### 更多视觉差异

- 不同类目用不同珊瑚模型（需要额外 glb 文件）
- Shape Keys 做枝干生长动画（需要建模）
- 每株珊瑚周围独立的彩色粒子
- 分层辉光，只让发光部分 bloom

### 更多交互

- 悬停预览
- 点进去看子分类
- 截图导出
