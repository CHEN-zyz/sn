# Coralithm — 设计文档

## 概念

Coralithm 把一个人的兴趣变成一片可以持续生长的 3D 珊瑚礁。

用户可以上传 YouTube 观看记录，也可以回答一组简短问题。两种方式都会得到兴趣类别及其占比，再由这些数据决定珊瑚的颜色、形态、亮度和运动。每次完成问答都能添加一株新珊瑚，礁群会随着使用逐渐扩大。

---

## 当前实现

### 两种数据入口

**YouTube 数据上传**

- 读取 Google Takeout 导出的 `watch-history.json`
- 全程在浏览器内解析，不把观看记录上传到服务器
- 提取视频标题、频道名和观看时间
- 通过韩文、英文关键词匹配，将记录分到 14 个主题类别
- 按类别统计观看次数、频道数量、最近观看时间和趋势
- 取观看量最高的 10 个类别生成珊瑚
- 上传新文件时，现有礁群会被替换

**随机问题模式**

- 题库共 14 道题，每次不重复地随机抽取 3 道
- 每道题有 4 个选项，每个选项对应一个兴趣类别
- 行为和具体偏好类问题权重较高，抽象感受类问题权重较低
- 同一类别被多次命中时分数累加
- 最终按各类别得分占比生成一株混合兴趣珊瑚
- 可以反复答题并继续添加珊瑚，不会清空已有礁群
- 名称由命中的类别自动组合，例如“游戏·音乐 珊瑚”

目前仍保留一个演示用的手动入口：用户按顺序选择 3 个类别，分别赋予 50%、30%、20% 的固定权重。

### 类别

当前共有 14 个主题类别：

新闻·时事、经济·投资、料理、游戏、体育、软件·AI、环境·气候、广告·营销、音乐、设计·艺术、旅行、灵感·洞察、学习、风格。

无法匹配的 YouTube 记录会归入“其他”。

### 数据与视觉

项目加载 7 个珊瑚模型，并按主类别循环分配。类别数量多于模型数量，因此不同类别仍可能共用同一种基础模型。

| 数据 | 视觉表现 |
|---|---|
| 主类别 | 珊瑚颜色与基础模型 |
| 类别占比 | 高度、透明度和轮廓变形幅度 |
| 频道多样性 | 横向粗细与材质粗糙度 |
| 最近活跃度 | 自发光强度与呼吸速度 |
| 趋势 | 自转速度 |
| 命中的多个类别 | 详情中的比例列表与组合名称 |

创建珊瑚时会用 simplex noise 做一次顶点变形。新珊瑚从接近零的尺寸弹性生长，并自动寻找空位，避免与已有珊瑚重叠。

### 问题模式中的模拟数据

问题答案只能推断兴趣类别和相对占比，不能得出真实的观看次数、频道多样性或最近活跃度。

当前实现为了驱动视觉效果，会根据答题结果生成一组模拟值：

- 观看次数由最高类别占比换算
- 频道多样性由命中类别数量推算
- 最近活跃度由最高类别占比推算
- 趋势使用固定值

详情面板目前没有区分这些模拟值和 YouTube 上传得到的真实统计，容易让用户误以为两者来源相同。正式版本需要决定是隐藏这些字段、标注为“推测”，还是为问题模式设计另一套更合适的指标。

### 场景与交互

- 黑色水下场景，带辉光、指数雾和暗角
- 3000 个漂流粒子、800 个海雪粒子和 3 道光束
- 拖拽旋转视角，滚轮缩放
- 悬停珊瑚时强化标签
- 点击珊瑚后相机移近，其他珊瑚淡出，并显示详情面板
- 点击空白处返回全景
- 可以单独删除珊瑚
- 每株珊瑚连接到最近的两个邻居，增删后自动重建连接线
- 支持将当前画面导出为 PNG
- “?”按钮提供 Google Takeout 导出说明

## 已知限制

### YouTube 分类仍较粗

目前只根据标题做关键词匹配。标题含义模糊、关键词缺失或跨类别时，分类结果可能不准确。“其他”类别也可能吸收较多无法识别的记录。

### 趋势指标还不成熟

代码已经保留趋势字段并将它映射到旋转速度，但目前的计算方式不足以真实表达兴趣随时间的变化，需要重新定义时间窗口和比较方法。

### 两种数据模式的语义不同

YouTube 模式描述的是实际观看行为；问题模式描述的是自我选择和即时偏好。它们可以使用同一套珊瑚语言，但详情面板不应把两类数据当成同一种测量结果。

### 模型与类别不是一一对应

目前是 14 个类别共用 7 个模型。类别主要依靠颜色区分，尚未形成完整的类别造型体系。

## 后续方向

### 先解决数据表达

- 明确标出真实统计与问答推测
- 为问题模式设计“兴趣强度”“偏好集中度”等更贴切的字段
- 决定组合珊瑚的命名方式，避免长期停留在类别拼接
- 重新设计趋势算法

### 提高分类质量

1. 从 Takeout URL 提取视频 ID，通过 YouTube API 获取 `categoryId`
2. 保留现有关键词规则作为兜底
3. 对模糊标题做批量语义分类

### 丰富珊瑚差异

- 扩充模型，使重点类别拥有更明确的轮廓
- 使用 Shape Keys 或骨骼制作枝干生长
- 增加每株珊瑚独有的粒子和局部辉光
- 支持查看类别构成和来源说明
